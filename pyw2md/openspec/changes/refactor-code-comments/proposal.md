## Why
当前项目代码中存在大量冗余、低质量且不一致的注释，违反了"好品味"的代码原则。这些注释不仅降低了代码可读性，还增加了维护负担。需要建立统一的注释规范，清理无用注释，提升代码质量。

## What Changes
- 建立基于Linus实用主义原则的中文注释规范
- 删除所有显而易见的函数文档字符串和行内注释
- 统一注释风格为简体中文，保持语言一致性
- 重构需要依赖注释才能理解的复杂代码
- 优化函数和变量命名，提升代码自解释能力

## Impact
- **Affected specs**: 新增 `code-quality` 规范
- **Affected code**: 所有Python源文件，特别是 `core/`, `utils/`, `ui/` 模块
- **Breaking changes**: 无，仅删除和优化注释，不改变API
- **Benefits**: 提升代码可读性，减少维护负担，符合"好品味"原则
